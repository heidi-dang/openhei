CLI openhei по умолчанию запускает [TUI](/docs/tui) при запуске без каких-либо аргументов.

```bash
openhei
```

Но он также принимает команды, описанные на этой странице. Это позволяет вам программно взаимодействовать с openhei.

```bash
openhei run "Explain how closures work in JavaScript"
```

---

### tui

Запустите TUI openhei.

```bash
openhei [project]
```

#### Флаги

| Флаг         | Короткий | Описание                                                                      |
| ------------ | -------- | ----------------------------------------------------------------------------- |
| `--continue` | `-c`     | Продолжить последний сеанс                                                    |
| `--session`  | `-s`     | Идентификатор сеанса для продолжения                                          |
| `--fork`     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`) |
| `--prompt`   |          | Промпт для использования                                                      |
| `--model`    | `-m`     | Модель для использования в виде поставщика/модели.                            |
| `--agent`    |          | Агент для использования                                                       |
| `--port`     |          | Порт для прослушивания                                                        |
| `--hostname` |          | Имя хоста для прослушивания                                                   |

## Команды

CLI openhei также имеет следующие команды.

---

### agent

Управляйте агентами для openhei.

```bash
openhei agent [command]
```

---

### attach

Подключите терминал к уже работающему внутреннему серверу openhei, запущенному с помощью команд `serve` или `web`.

```bash
openhei attach [url]
```

Это позволяет использовать TUI с удаленным сервером openhei. Например:

```bash
# Start the backend server for web/mobile access
openhei web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
openhei attach http://10.20.30.40:4096
```

#### Флаги

| Флаг        | Короткий | Описание                             |
| ----------- | -------- | ------------------------------------ |
| `--dir`     |          | Рабочий каталог для запуска TUI      |
| `--session` | `-s`     | Идентификатор сеанса для продолжения |

### auth

Команда для управления учетными данными и входом в систему для провайдеров.

```bash
openhei auth [command]
```

---

#### login

openhei использует список провайдеров с [Models.dev](https://models.dev), поэтому вы можете использовать `openhei auth login` для настройки ключей API для любого поставщика, которого вы хотите использовать. Это хранится в `~/.local/share/openhei/auth.json`.

```bash
openhei auth login
```

Когда openhei запускается, он загружает поставщиков из файла учетных данных. И если в ваших средах определены какие-либо ключи или файл `.env` в вашем проекте.

---

#### list

Перечисляет всех проверенных поставщиков, которые хранятся в файле учетных данных.

```bash
openhei auth list
```

Или короткая версия.

```bash
openhei auth ls
```

---

#### logout

Выключает вас из провайдера, удаляя его из файла учетных данных.

```bash
openhei auth logout
```

#### add

Или используйте короткую версию.

```bash
openhei mcp ls
```

#### auth

Аутентификация с помощью сервера MCP с поддержкой OAuth.

```bash
openhei mcp auth [name]
```

Если вы не укажете имя сервера, вам будет предложено выбрать один из доступных серверов с поддержкой OAuth.

Вы также можете перечислить серверы с поддержкой OAuth и их статус аутентификации.

```bash
openhei mcp auth list
```

Или используйте короткую версию.

```bash
openhei mcp auth ls
```

---

#### logout

Удалите учетные данные OAuth для сервера MCP.

```bash
openhei mcp logout [name]
```

---

#### debug

Отладка проблем с подключением OAuth для сервера MCP.

```bash
openhei mcp debug <name>
```

### models

Перечислите все доступные модели от настроенных поставщиков.

```bash
openhei models [provider]
```

Эта команда отображает все модели, доступные у настроенных вами поставщиков, в формате `provider/model`.

Это полезно для определения точного названия модели, которое будет использоваться в [вашем config](/docs/config/).

При желании вы можете передать идентификатор поставщика, чтобы фильтровать модели по этому поставщику.

```bash
openhei models anthropic
```

#### Флаги

| Флаг        | Описание                                                                          |
| ----------- | --------------------------------------------------------------------------------- |
| `--refresh` | Обновите кеш моделей на сайте models.dev.                                         |
| `--verbose` | Используйте более подробный вывод модели (включая метаданные, такие как затраты). |

Используйте флаг `--refresh` для обновления списка кэшированных моделей. Это полезно, когда к поставщику добавлены новые модели и вы хотите увидеть их в openhei.

```bash
openhei models --refresh
```

### run

Запустите openhei в неинтерактивном режиме, передав приглашение напрямую.

```bash
openhei run [message..]
```

Это полезно для создания сценариев, автоматизации или когда вам нужен быстрый ответ без запуска полного TUI. Например.

```bash "openhei run"
openhei run Explain the use of context in Go
```

Вы также можете подключиться к работающему экземпляру `openhei serve`, чтобы избежать холодной загрузки сервера MCP при каждом запуске:

```bash
# Start a headless server in one terminal
openhei serve

# In another terminal, run commands that attach to it
openhei run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Флаги

| Флаг         | Короткий | Описание                                                                         |
| ------------ | -------- | -------------------------------------------------------------------------------- |
| `--command`  |          | Команда для запуска, используйте сообщение для аргументов                        |
| `--continue` | `-c`     | Продолжить последний сеанс                                                       |
| `--session`  | `-s`     | Идентификатор сеанса для продолжения                                             |
| `--fork`     |          | Разветвить сеанс при продолжении (используйте с `--continue` или `--session`)    |
| `--share`    |          | Поделиться сеансом                                                               |
| `--model`    | `-m`     | Модель для использования в виде поставщика/модели.                               |
| `--agent`    |          | Агент для использования                                                          |
| `--file`     | `-f`     | Файл(ы) для прикрепления к сообщению                                             |
| `--format`   |          | Формат: по умолчанию (отформатированный) или json (необработанные события JSON). |
| `--title`    |          | Название сеанса (использует усеченное приглашение, если значение не указано)     |
| `--attach`   |          | Подключитесь к работающему серверу openhei (например, http://localhost:4096)    |
| `--port`     |          | Порт локального сервера (по умолчанию случайный порт)                            |

### serve

Запустите автономный сервер openhei для доступа к API. Полный HTTP-интерфейс можно найти в [server docs](/docs/server).

```bash
openhei serve
```

При этом запускается HTTP-сервер, который обеспечивает доступ API к функциям openhei без интерфейса TUI. Установите `OPENHEI_SERVER_PASSWORD`, чтобы включить базовую аутентификацию HTTP (имя пользователя по умолчанию — `openhei`).

#### Флаги

| Флаг         | Описание                                                      |
| ------------ | ------------------------------------------------------------- |
| `--port`     | Порт для прослушивания                                        |
| `--hostname` | Имя хоста для прослушивания                                   |
| `--mdns`     | Включить обнаружение mDNS                                     |
| `--cors`     | Дополнительные источники браузера, позволяющие разрешить CORS |

### stats

Покажите статистику использования токенов и затрат для ваших сеансов openhei.

```bash
openhei stats
```

#### Флаги

| Флаг        | Описание                                                                                                   |
| ----------- | ---------------------------------------------------------------------------------------------------------- |
| `--days`    | Показать статистику за последние N дней (все время)                                                        |
| `--tools`   | Количество инструментов для отображения (все)                                                              |
| `--models`  | Показать разбивку по использованию модели (по умолчанию скрыто). Передайте номер, чтобы показать верхнюю N |
| `--project` | Фильтровать по проекту (все проекты, пустая строка: текущий проект)                                        |

### export

Экспортируйте данные сеанса в формате JSON.

```bash
openhei export [sessionID]
```

Если вы не укажете идентификатор сеанса, вам будет предложено выбрать один из доступных сеансов.

### import

Импортируйте данные сеанса из файла JSON или URL-адреса общего ресурса openhei.

```bash
openhei import <file>
```

Вы можете импортировать из локального файла или URL-адреса общего ресурса openhei.

```bash
openhei import session.json
openhei import https://opncd.ai/s/abc123
```

### acp

Запустите сервер ACP (агент-клиентский протокол).

```bash
openhei acp
```

Эта команда запускает сервер ACP, который обменивается данными через stdin/stdout с использованием nd-JSON.

#### Флаги

| Флаг         | Описание                    |
| ------------ | --------------------------- |
| `--cwd`      | Рабочий каталог             |
| `--port`     | Порт для прослушивания      |
| `--hostname` | Имя хоста для прослушивания |

---

### uninstall

Удалите openhei и удалите все связанные файлы.

```bash
openhei uninstall
```

#### Флаги

| Флаг            | Короткий | Описание                                   |
| --------------- | -------- | ------------------------------------------ |
| `--keep-config` | `-c`     | Сохраняйте файлы конфигурации              |
| `--keep-data`   | `-d`     | Храните данные сеанса и снимки             |
| `--dry-run`     |          | Покажите, что было бы удалено без удаления |
| `--force`       | `-f`     | Пропустить запросы подтверждения           |

---

### upgrade

Обновляет openhei до последней версии или определенной версии.

```bash
openhei upgrade [target]
```

Чтобы обновиться до последней версии.

```bash
openhei upgrade
```

Для обновления до определенной версии.

```bash
openhei upgrade v0.1.48
```

#### Флаги

| Флаг       | Короткий | Описание                                                  |
| ---------- | -------- | --------------------------------------------------------- |
| `--method` | `-m`     | Используемый метод установки: local, npm, pnpm, bun, brew |

| `OPENHEI_CLIENT`                     | строка              | Идентификатор клиента (по умолчанию `cli`)                                       |
