Anche se OpenHei puo essere eseguito direttamente su Windows, consigliamo di usare [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) per la migliore esperienza. WSL fornisce un ambiente Linux che funziona in modo ottimale con le funzionalita di OpenHei.

:::tip[Perche WSL?]
WSL offre prestazioni migliori del file system, supporto completo del terminale e compatibilita con gli strumenti di sviluppo su cui OpenHei fa affidamento.
:::

---

## Configurazione

<Steps>

1. **Installa WSL**

   Se non l'hai ancora fatto, [installa WSL](https://learn.microsoft.com/en-us/windows/wsl/install) seguendo la guida ufficiale Microsoft.

2. **Installa OpenHei in WSL**

   Dopo aver configurato WSL, apri il terminale WSL e installa OpenHei usando uno dei [metodi di installazione](/docs/).

   ```bash
   bash -c "$(curl -fsSL https://raw.githubusercontent.com/heidi-dang/openhei/main/install.sh)"
   ```

3. **Usa OpenHei da WSL**

   Vai nella directory del progetto (accedi ai file Windows tramite `/mnt/c/`, `/mnt/d/`, ecc.) ed esegui OpenHei.

   ```bash
   cd /mnt/c/Users/YourName/project
   openhei
   ```

</Steps>

## App desktop + server WSL

Se preferisci usare l'app desktop di OpenHei ma vuoi eseguire il server in WSL:

1. **Avvia il server in WSL** con `--hostname 0.0.0.0` per consentire connessioni esterne:

   ```bash
   openhei serve --hostname 0.0.0.0 --port 4096
   ```

2. **Collega l'app desktop** a `http://localhost:4096`

:::note
Se `localhost` non funziona nella tua configurazione, usa l'indirizzo IP di WSL (da WSL: `hostname -I`) e collega `http://<wsl-ip>:4096`.
:::

:::caution
Quando usi `--hostname 0.0.0.0`, imposta `OPENHEI_SERVER_PASSWORD` per proteggere il server.

```bash
OPENHEI_SERVER_PASSWORD=your-password openhei serve --hostname 0.0.0.0
```

:::

## Client web + WSL

Per la migliore esperienza web su Windows:

1. **Esegui `openhei web` nel terminale WSL** invece che in PowerShell:

   ```bash
   openhei web --hostname 0.0.0.0
   ```

2. **Accedi dal browser Windows** su `http://localhost:<port>` (OpenHei stampa l'URL)

Eseguire `openhei web` da WSL garantisce accesso corretto al file system e integrazione del terminale, restando comunque accessibile dal browser Windows.

## Accesso ai file Windows

WSL puo accedere a tutti i file Windows tramite la directory `/mnt/`:

- Unita `C:` → `/mnt/c/`
- Unita `D:` → `/mnt/d/`
- E cosi via...

Esempio:

```bash
cd /mnt/c/Users/YourName/Documents/project
openhei
```

:::tip
Per un'esperienza piu fluida, valuta di clonare o copiare il repository nel file system di WSL (per esempio in `~/code/`) ed eseguire OpenHei li.
:::

## Suggerimenti

- Tieni OpenHei in esecuzione su WSL per i progetti salvati su unita Windows: l'accesso ai file e immediato
- Usa l'[estensione WSL di VS Code](https://code.visualstudio.com/docs/remote/wsl) insieme a OpenHei per un flusso di lavoro integrato
- Configurazione e sessioni di OpenHei sono salvate nell'ambiente WSL in `~/.local/share/openhei/`
